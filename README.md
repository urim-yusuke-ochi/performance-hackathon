# Web Speed Hackathon 2025 AREMA

## 概要

**"Web Speed Hackathon 2025" は、非常に重たい Web アプリをチューニングして、いかに高速にするかを競う競技です。**

今回のテーマは、架空の動画配信サービス「AREMA」です。
レギュレーションを守った上で、AREMA のパフォーマンスを改善してください。

## 社内実施

本リポジトリを使った社内パフォーマンスチューニング研修の運営手順です。

### アーキテクチャ

```
参加者リポジトリ (main push)
    │
    ▼
GitHub Actions
    ├── Docker Build → ECR
    ├── Deploy → ECS Fargate
    └── Lighthouse 計測 → S3
                              │
                              ▼
CloudFront → ALB → ECS (参加者ごと)
S3 → Dashboard (全員のスコア一覧)
```

### 管理者の準備手順

#### 1. AWSインフラのデプロイ（研修数日前）

```bash
cd infra/scripts

export AWS_REGION=ap-northeast-1
export DOMAIN_NAME="performance-hackathon.your domain"
export CERTIFICATE_ARN="証明書arn"

./setup.sh
```

> ACM証明書はワイルドカード (`*.performance-hackathon.snd-dev.urim-aws.com`) で us-east-1 に事前作成が必要です。

#### 2. DNS設定

| レコード | 向き先 |
|---------|-------|
| `*.performance-hackathon.snd-dev.urim-aws.com` | CloudFront Distribution |
| `dashboard.performance-hackathon.snd-dev.urim-aws.com` | Dashboard CloudFront |

#### 3. 参加者リポジトリの作成（研修前日〜当日）

GitHub上で本リポジトリを **テンプレートリポジトリ** に設定し、参加者ごとに `Use this template` で個人リポジトリを作成してもらう。

> fork でも可能ですが、fork の場合は参加者が Actions タブで手動有効化が必要です。テンプレート方式の方が確実です。

#### 4. 参加者の登録 & Secrets 設定

```bash
# 参加者リストを作成（Github nameを記載）
cat > participants.txt << EOF
aaaa
bbbb
EOF

# 一括でターゲットグループ作成 + GitHub Secrets設定
./bulk-setup-secrets.sh participants.txt
```

個別追加する場合：

```bash
./add-participant.sh <github-username> <priority>
```

#### 5. ダッシュボードのデプロイ

GitHub Actions の「Deploy Dashboard」ワークフローを手動実行、もしくは：

```bash
aws s3 sync dashboard/ s3://performance-hackathon-dashboard/ --delete
```

#### 6. 研修終了後のクリーンアップ

```bash
# ECSサービス全削除
aws ecs list-services --cluster performance-hackathon --query 'serviceArns[]' --output text | \
  xargs -n1 aws ecs delete-service --cluster performance-hackathon --force --service

# CloudFormationスタック削除
aws s3 rm s3://performance-hackathon-scores --recursive
aws s3 rm s3://performance-hackathon-dashboard --recursive
aws cloudformation delete-stack --stack-name performance-hackathon-base
```

### 参加者の作業

1. テンプレートから自分のリポジトリを作成
2. ローカルで `pnpm install` → `pnpm run start` で動作確認
3. パフォーマンス改善を実施
4. `main` ブランチに push → 自動デプロイ＆計測
5. ダッシュボード (`https://dashboard.performance-hackathon.your-domain.com`) でスコア確認

### GitHub Secrets 一覧（管理者が設定）

| Secret | 説明 |
|--------|------|
| `AWS_ROLE_ARN` | GitHub Actions 用 IAM Role ARN |
| `ECS_EXECUTION_ROLE_ARN` | ECS タスク実行ロール |
| `ECS_TASK_ROLE_ARN` | ECS タスクロール |
| `CLOUDFRONT_DISTRIBUTION_ID` | CloudFront ディストリビューション ID |
| `SUBNET_IDS` | ECS 用サブネット ID (カンマ区切り) |
| `SECURITY_GROUP_ID` | ECS 用セキュリティグループ ID |
| `TARGET_GROUP_ARN_PREFIX` | 参加者ごとの ALB ターゲットグループ ARN |
